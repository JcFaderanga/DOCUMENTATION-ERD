RUBRICS

Advanced Database System Final Project

Component Points Description
Business Rules 10 points
  • Clearly defined and documented business rules relevant to the project.
  • Business rules align with the project's objectives and requirements.
  • Business rules address data integrity, validation, and constraints.
Entity-Relationship Diagram (ERD) 15 points
  • Well-designed ERD illustrating entities, attributes, and relationships.
  • Appropriate use of primary keys, foreign keys, and relationships.
  • ERD accurately represents the project's data model and relationships.
User Interface 10 points
  • Intuitive and easy-to-understand user interface.
  • User interface design enhances user experience and usability.
Database Design 15 points
  • Database design matches the ERD and business rules.
  • Proper table structures, relationships, and data types are used.
  • Database design supports efficient data retrieval and manipulation.
Functionalities 50 points a) User Levels (10 points)
  • Implementation of multiple user levels with appropriate access controls.
  • Clearly defined roles and permissions for each user level.
  • Access restrictions and security measures are in place.
b) Reporting (15 points)
  • Integration of reporting functionalities for data analysis and presentation.
  • Availability of predefined reports and ability to generate custom reports.
  • Reports provide relevant and accurate information.
c) Logs (5 points)
  • System logs capture relevant user activities and system events.
  • Logs include timestamps, user identification, and details of actions performed.
  • Logs are accessible and provide a comprehensive audit trail.
d) Stored Procedures (10 points)
  • Effective utilization of stored procedures for complex data manipulation.
  • Stored procedures improve system performance and maintain data integrity.
  • Procedures are well-documented and adhere to best practices.
e) Encryption and Decryption (10 points)
  • Appropriate implementation of encryption techniques to protect sensitive data.
  • Encryption methods are robust and secure against common vulnerabilities.
  • Decryption processes ensure authorized access to encrypted data.
Documentation and Presentation 10 points
  • Well-structured and comprehensive documentation of the system.
  • Documentation includes system architecture, data dictionaries, and user manuals.
  • Effective and professional presentation of the final system development.
